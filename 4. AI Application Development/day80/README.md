# 80일차 (2026-07-07)

## 학습 주제

4차 프로젝트의 사용자 인증 기능과 실작명 API 연동 기반을 정리했다.

## 핵심 작업

### 1. Django 사용자 인증 백엔드

- 세션 기반 CSRF, 로그인, 로그아웃, 회원가입 API를 연결했다.
- 이메일 중복 확인, 아이디 찾기, 비밀번호 재설정 흐름을 추가했다.
- 회원가입 시 약관 및 개인정보처리방침 동의 정보를 `UserConsent`로 저장하도록 모델과 마이그레이션을 구성했다.
- 로그인 상태와 계정 상태를 확인하는 보호 API 및 마이페이지 조회 흐름을 정리했다.

### 2. React 인증 흐름

- 프론트엔드 인증 API 클라이언트와 `AuthProvider`를 연결했다.
- 새로고침 후 세션을 복원하고, 인증이 필요한 화면에 접근할 때 로그인 화면으로 이동하도록 라우팅을 구성했다.
- 로그인, 회원가입, 비밀번호 재설정 화면을 실제 Django API와 연동했다.
- GNB와 마이페이지에 계정관리, 비밀번호 변경, 회원 탈퇴 흐름을 반영했다.

### 3. 약관 화면과 사용자 안내 정리

- 이용약관, 개인정보처리방침, FAQ 문구를 실제 서비스 흐름에 맞게 수정했다.
- 약관 동의 모달과 정책 상세 화면의 불필요한 내용을 정리했다.
- 인증 연동 작업 내역과 팀원 인계 내용을 문서화했다.

### 4. 실작명 백엔드 연동

- Mock으로 고정되어 있던 이름 결과를 FastAPI 작명 API 호출 방식으로 전환했다.
- constraint-first 방식으로 입력 조건을 우선 적용하도록 작명 전략을 보완했다.
- 성씨, 성별, 오행, 이름 글자 수 등의 조건을 자연어 입력과 연결했다.
- 다시 추천할 때 기존 결과를 `excludeNames`로 전달해 같은 이름이 반복되지 않도록 구성했다.

- 인증 백엔드: `webapp/naming/views.py`, `webapp/naming/forms.py`, `webapp/naming/models.py`
- 인증 프론트엔드: `frontend/src/api/auth.ts`, `frontend/src/app/providers/AuthProvider.tsx`
- 작명 연동: `fastapi_app/main.py`, `src/graph/naming_graph.py`
- 작업 문서: `docs/작업내역_지용_로그인_회원가입_마이페이지_1차.md`

## 공개 정리 기준

Daily에는 전체 프로젝트 코드와 환경 파일을 복사하지 않고 핵심 작업 내용만 기록한다. `.env`, API 키, 비밀번호, ChromaDB와 같은 실행 환경 자료는 포함하지 않는다.

## 한 줄 요약

사용자 인증을 실제 Django 세션 API와 연결하고, Mock 작명 결과를 조건 기반 실작명 API 흐름으로 전환한 날이다.
