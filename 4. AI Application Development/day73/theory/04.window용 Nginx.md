# Windows용 Nginx 리버스 프록시 연동

본 단계에서는 Windows 개발 환경에서 사용할 수 있는 Windows용 Nginx 웹 서버를 다운로드하여 구성하고, WAS(Waitress)와 TCP 소켓 통신을 매개로 삼아 리버스 프록시(Reverse Proxy)로 연동하는 방법을 학습합니다.

---

## 1. Windows Nginx의 특징과 통신 제약

Nginx는 본래 Linux/UNIX 환경에 적합하도록 구현되어 대규모 트래픽 처리에 압도적인 성능을 보입니다. Windows 버전의 Nginx 역시 배포되어 개발 및 간단한 테스트에 널리 사용되지만, 몇 가지 플랫폼 고유의 한계와 제약이 있습니다.

- **UNIX 도메인 소켓 지원 불가**: Windows 운영체제 커널의 구조적 특성상, 파일 상의 소켓인 UNIX 소켓 통신을 Nginx Windows 포팅 버전에서 사용할 수 없습니다.
- **TCP 루프백 통신 대체**: 따라서 Windows 환경에서 Nginx와 WAS(Waitress)를 중계하려면 반드시 로컬 호스트 IP인 `127.0.0.1`의 특정 TCP 포트(예: 8000)를 사용해 상호 통신해야 합니다.

---

## 2. Nginx와 WAS(Waitress) 연동 흐름

```text
[클라이언트 브라우저]
       │  (HTTP: localhost)
       ▼
 [Nginx (Windows)]  <─── 포트 80에서 대기
       │
       │  (TCP Proxy: 127.0.0.1:8000)
       ▼
[Waitress (Windows)] <─── 포트 8000에서 대기 (Django 실행)
```

## 3. 현재 PC 기준 구성 경로

강의 원본 예시는 `C:\nginx`를 기준으로 설명하지만, 현재 PC에서는 압축 해제 폴더명이 다음과 같습니다.

```text
C:\nginx-1.30.3
```

현재 프로젝트와 Nginx 설정 연결 구조는 다음과 같습니다.

```text
C:\python-src\Daily\4. AI Application Development\
├── day73\                         # Django 웹 소스 코드 및 Waitress 실행 코드
└── nginx\
    └── django.conf                # Nginx http 블록 안에서 include되는 Django 프록시 설정

C:\nginx-1.30.3\
└── conf\
    └── nginx.conf                 # 실제 Nginx 실행 설정 파일
```

현재 PC에서는 `C:\nginx-1.30.3\conf\nginx.conf`의 `http { ... }` 블록 안에 아래 include가 연결되어 있습니다.

```nginx
include "C:/python-src/Daily/4. AI Application Development/nginx/django.conf";
```

따라서 `4. AI Application Development/nginx/django.conf` 파일은 `worker_processes`, `events`, `http` 전체를 포함하는 완성형 설정 파일이 아니라, `http` 블록 내부에 들어가는 조각(fragment) 설정 파일로 유지해야 합니다.

## 4. Nginx 프록시 설정 파일

- 파일명: `C:\python-src\Daily\4. AI Application Development\nginx\django.conf`

```nginx
# upstream 정의: Windows 환경에서 기동 중인 Waitress TCP 주소(127.0.0.1:8000)를 연결합니다.
# Nginx Windows 버전은 UNIX 도메인 소켓 방식을 지원하지 않으므로 TCP 방식을 사용합니다.
upstream django_app {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name localhost;

    # 파일 업로드 크기 제한 해제 (50MB)
    client_max_body_size 50M;

    # Gzip 압축 전송 설정
    gzip on;
    gzip_comp_level 5;
    gzip_min_length 1024;
    gzip_proxied any;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 로깅 파일 설정 (Windows Nginx의 기본 logs 폴더 내에 저장)
    access_log logs/django_access.log;
    error_log logs/django_error.log;

    # 정적 파일 직접 서빙
    location /static/ {
        alias "C:/python-src/Daily/4. AI Application Development/day73/staticfiles/";
        expires 30d;
        add_header Cache-Control "public, no-transform";
        access_log off;
    }

    # 미디어 파일 직접 서빙
    location /media/ {
        alias "C:/python-src/Daily/4. AI Application Development/day73/media/";
        expires 7d;
        add_header Cache-Control "public, no-transform";
    }

    # 모든 일반 동적 요청을 Waitress WAS로 프록싱
    location / {
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Host $http_host;

        # 리다이렉트 충돌 방지
        proxy_redirect off;

        # 정의된 TCP upstream으로 패스
        proxy_pass http://django_app;
    }
}
```

로그 파일은 프로젝트 폴더가 아니라 Nginx 실행 폴더 기준으로 생성됩니다.

```text
C:\nginx-1.30.3\logs\django_access.log
C:\nginx-1.30.3\logs\django_error.log
```

강의 원본에 새로 추가된 전체 설정 파일도 현재 PC 경로에 맞춰 보관되어 있습니다.

```text
C:\python-src\Daily\4. AI Application Development\nginx\nginx.conf
```

이 파일은 `C:\nginx-1.30.3\conf\nginx.conf` 전체를 교체하고 싶을 때 참고용으로 사용할 수 있습니다. 현재 PC에서는 기존 `C:\nginx-1.30.3\conf\nginx.conf`에서 `django.conf`를 include하는 방식으로 이미 연결되어 있으므로, 별도로 전체 교체하지 않아도 됩니다.

## 5. Waitress 실행 스크립트 확인

4단계에서 고도화한 `run_waitress.py` 코드가 그대로 유지되는지 점검합니다. Nginx가 아직 정적 파일을 직접 처리하지 않으므로, Waitress가 정적/미디어 서빙을 겸할 수 있도록 래핑된 핸들러를 사용합니다.

- 파일명: `C:\python-src\Daily\4. AI Application Development\day73\run_waitress.py`

현재 로컬 실습 환경에서는 PostgreSQL 운영 DB 설정이 완전히 준비되기 전에도 실행할 수 있도록 기본값이 `todoboard.settings.local`입니다. 운영 DB까지 연결해서 실행하려면 실행 전에 환경 변수를 `todoboard.settings.production`으로 지정합니다.

```powershell
# 로컬 SQLite 기준 실행
python run_waitress.py

# PostgreSQL production 설정 기준 실행
$env:DJANGO_SETTINGS_MODULE='todoboard.settings.production'
python run_waitress.py
```

## 6. Windows에서 Nginx 기동 및 연동 테스트

Windows 환경에서 Nginx를 다운로드하여 설정 파일을 적용하고 구동하는 절차는 다음과 같습니다.

1. **Nginx Windows 버전 다운로드**: [Nginx 공식 홈페이지](https://nginx.org/en/download.html)의 Stable version 메뉴에서 `nginx/Windows-x.xx.x.zip` 압축 파일을 다운로드합니다. 현재 PC에서는 `C:\nginx-1.30.3` 경로에 압축이 해제되어 있습니다.
2. **설정 파일 연결 확인**: `C:\nginx-1.30.3\conf\nginx.conf`의 `http` 블록 안에 `4. AI Application Development/nginx/django.conf` include가 들어있는지 확인합니다.
3. **Waitress 및 Nginx 구동**:

   ```powershell
   # 1. Django 프로젝트 폴더로 이동
   cd "C:\python-src\Daily\4. AI Application Development\day73"

   # 2. 로컬 DB 기준 기본 점검
   python manage.py check

   # 3. PostgreSQL production 설정을 사용할 경우 먼저 마이그레이션 적용
   python manage.py migrate --settings=todoboard.settings.production

   # 4. Nginx가 직접 서빙할 정적 파일 수집
   python manage.py collectstatic --noinput

   # 5. Django 가상환경 터미널에서 Waitress 기동
   python run_waitress.py

   # 6. 다른 터미널에서 Nginx 설정 검사 및 실행
   cd C:\nginx-1.30.3
   .\nginx.exe -t -p C:\nginx-1.30.3\
   .\nginx.exe
   ```

4. **연동 종료 및 제어**:

   ```powershell
   cd C:\nginx-1.30.3

   # 설정 변경 후 재적용
   .\nginx.exe -s reload

   # Nginx 안전하게 종료
   .\nginx.exe -s quit

   # Nginx 강제 종료
   .\nginx.exe -s stop
   ```

브라우저에서 `http://localhost` 주소로 포트 번호 없이 접속했을 때, Nginx가 요청을 수락하여 포트 8000번의 Waitress 서버로 동적 페이로드를 전달하고 Django 화면이 에러 없이 노출된다면 성공입니다.
